1. **YouTube Transcript API 최신 사용법**

2. **Claude API 구조화된 프롬프트의 힘**

   - 간단한 예제보다 실제 데이터(AI Memory 360 Tour)로 테스트

- 해결: `help(YouTubeTranscriptApi)`로 현재 메서드 직접 확인
- 교훈: 라이브러리는 항상 최신 문서와 실제 코드 확인 필요

### 가장 유용한 부분

**90% 시간 절약 달성**

## 3. 발생한 문제와 해결

### 문제 1: YouTube Transcript API 메서드 변경

| 항목 | 내용 |
|------|------|
| **증상** | `AttributeError: 'get_transcript' not found` |
| **원인** | 라이브러리 최신 버전에서 API 구조 변경 |
| **시도** | 1) 공식 문서 확인 (구버전 정보) 2) GitHub Issues 검색 |
| **해결** | `help(YouTubeTranscriptApi)` 실행하여 실제 메서드 확인 |
| **소요 시간** | 20분 |
| **교훈** | 라이브러리 오류 시 help() 명령어가 가장 빠른 해결책 |

### 문제 2: Windows 콘솔 UTF-8 인코딩

| 항목 | 내용 |
|------|------|
| **증상** | `UnicodeEncodeError: 'charmap' codec can't encode` |
| **원인** | Windows 콘솔 기본 인코딩 cp1252 (이모지/한글 미지원) |
| **해결** | `sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')` |
| **소요 시간** | 10분 |
| **교훈** | Windows 스크립트 작성 시 인코딩 설정 필수 |

### 문제 3: Claude API 모델명 오류

| 항목 | 내용 |
|------|------|
| **증상** | `Error 404: model claude-3-5-sonnet-20241022 not found` |
| **원인** | Claude API 모델명 업데이트 |
| **해결** | `claude-sonnet-4-5-20250929`로 변경 |
| **소요 시간** | 5분 |
| **교훈** | 모델명은 변수로 분리하여 쉽게 업데이트 가능하게 설계 |
